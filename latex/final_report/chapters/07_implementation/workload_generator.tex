\section{Workload Generator}
\label{sec:workload_generator}
The Workload Generator is responsible for synthesizing a realistic arrival stream of Layer-2 transactions. It drives the synthetic load for all experiments in the benchmark suite, ensuring that each configuration is tested under identical traffic conditions.

\begin{itemize}
    \item \textbf{Language:} Python.
    \item \textbf{Generation Method:} It employs an Arrival Process Engine configured to generate steady-state traffic following a Poisson distribution. This accurately models independent user arrivals. Note that the generator currently focuses on steady-state and skewed transaction mixes but does not implement complex bursty transfer patterns.
    \item \textbf{Transaction Format:} The generator synthesizes JSON-RPC transaction requests. To ensure validity during high-throughput benchmarks without requiring expensive, compute-bound cryptographic signing operations from the load generator itself, the system uses a dummy signature approach. Specifically, the sequencer's \texttt{UserTransaction::hash()} function deliberately excludes \texttt{calldata} and \texttt{tx\_type} to allow these dummy signatures to pass verification.
    \item \textbf{Input/Output Files:} The generator interacts with the sequencer via an HTTP/RPC client. It tags transactions with Experiment IDs and Run IDs and outputs detailed telemetry, specifically \texttt{workload\_<exp\_id>.json} and \texttt{tx\_log\_<run\_id>.csv}, which record submission timestamps and latency.
\end{itemize}
