This chapter details the architecture and implementation of the modular ZK-rollup benchmarking prototype developed for this research project. The system separates sequencing, execution, proving, and submission stages to allow controlled configuration sweeps over specific bottlenecks. The core components are implemented in Rust using Domain-Driven Design (DDD) principles and SOLID architectural patterns. The system relies on containerized microservices (via Docker Compose) for orchestrated benchmarking.
