\subsection{RQ2: Impact of Batching}
The results clearly illustrate the core throughput--latency--cost trade-off. Optimizing for the absolute lowest Layer-1 cost per transaction dictates the use of exceptionally large batches. However, this inherently penalizes early-arriving users with high queueing latency. Conversely, fast soft finality requires smaller, rapidly sealed batches, which drives up the amortized L1 cost.
