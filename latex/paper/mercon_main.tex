\title{A Modular ZK-Rollup Prototype for Benchmarking Layer-2 Scalability Bottlenecks}

% Shared author block from root folder

\begin{abstract}
Ethereum's limited Layer-1 throughput has motivated the adoption of Layer-2 rollups, where ZK-Rollups improve scalability by executing transactions off-chain and submitting validity proofs or proof metadata to Layer-1. However, practical ZK-Rollup performance is shaped by interacting configuration choices, including batch formation, sequencing policy, data availability mode, Layer-1 submission cost, and proof-generation overhead. Existing production rollups are highly optimized for deployment, but their complexity makes controlled and reproducible experimentation difficult.

This paper presents a modular ZK-Rollup benchmarking prototype designed for controlled performance evaluation. The prototype separates sequencing, execution, proof handling, submission, and data availability concerns, enabling experiments over different batching, scheduling, and data-posting configurations. The current evaluation uses controlled synthetic workloads and parameterized proof-delay measurements to compare relative behavior across configurations using metrics such as committed throughput, latency, gas cost per transaction, batch finalization time, data availability footprint, and proof-delay contribution.

The preliminary results show how modular benchmarking can expose throughput--latency--cost trade-offs in the rollup pipeline without claiming production-level performance. The framework provides a reproducible basis for studying ZK-Rollup scalability bottlenecks and supports future extension toward full cryptographic prover benchmarking and trace-driven workloads.
\end{abstract}
