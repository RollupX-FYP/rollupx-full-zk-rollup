\section{Background and Related Work}

\subsection{Ethereum Scalability and Rollups}
Ethereum's rollup-centric roadmap relies on Layer-2 solutions to move execution off-chain while utilizing Layer-1 for settlement and data availability \cite{ethereum_scaling_docs}. ZK-Rollups use validity proof systems (such as Groth16 or PLONK) to mathematically guarantee off-chain execution correctness \cite{groth2016snark, ben2018stark}.

\subsection{Performance Bottlenecks and Benchmarking}
Recent benchmarking efforts highlight multiple bottlenecks in ZK-Rollups. Generating validity proofs remains a significant computational hurdle \cite{habib2025bottlenecks}. Furthermore, data availability poses a trade-off between cost and security; the introduction of EIP-4844 (proto-danksharding) aims to alleviate this by providing cheaper blob-based storage \cite{eip4844, saif2024dataavailability, lavaur2023modular}. 

EIP-4844 introduces blob-carrying transactions where large rollup data are made available through blob space rather than permanent calldata. The EVM cannot directly access blob contents, but it can access commitments to blob data, making blobs suitable for rollup data availability rather than contract execution data. Ethereum's data availability documentation also notes that after EIP-4844, some rollups post data to cheaper blob storage rather than permanent calldata, but blob data are only guaranteed for a fixed window \cite{ethereum_data_availability_docs}. This changes the cost structure of rollup submission and motivates evaluating calldata and blob-based DA separately.

The design of the sequencer and the chosen transaction ordering policies also heavily influence latency and fairness \cite{motepalli2023sequencers}. Existing studies often analyze public datasets or production traces \cite{silva2025zksyncdataset, chaliasos2024benchmarking}.

\subsection{Research Gap}
While prior work examines rollup architectures, costs, data availability, sequencer decentralization, and production traces, there is still a need for a compact modular framework that supports controlled experiments over batching, sequencing, DA, and proof overhead under identical workloads.
